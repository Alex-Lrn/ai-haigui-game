# AI 海龟汤游戏 PRD（Vibe Coding 实战）

## 1. 产品概述
本产品是一个 AI 驱动的海龟汤推理游戏网站。玩家选择某个海龟汤“故事”，进入游戏后与 AI 主持人对话，通过不断提问来逼近故事真相。  
AI 主持人只基于“汤底”给出有限回答：**是 / 否 / 无关**，不解释、不剧透，从而保证推理过程的公平性与神秘感。

## 2. 目标用户
- 喜欢推理与情境问答的玩家
- 想在浏览器快速娱乐的用户（碎片时间）
- 对 AI 应用开发感兴趣、希望理解“对话即开发”的技术爱好者

## 3. 产品目标（成功标准）
- 玩家可在 3 步内完成一次完整玩法：大厅选择故事 -> 游戏对话推理 -> 查看汤底揭晓
- AI 回复严格遵循三选一规则：**只能返回“是/否/无关”**（通过后端校验与兜底实现）
- 移动端可用：页面布局与聊天输入在手机上可读可操作
- 沉浸体验：加载态与错误态清晰，避免弹窗打断；整体风格统一为“科幻悬疑游戏官网风”

## 4. 核心功能

### 4.1 MVP（必须做）
1. **游戏大厅（Home）**
   - 展示 3-5 个海龟汤故事卡片（包含：标题、难度、入口按钮/点击进入）
   - 点击卡片进入对应游戏页面（路由形如：`/game/:id`，后续可实现）

2. **游戏页面（Game）**
   - 展示当前故事的 **汤面（surface）** 与故事基础信息（标题、难度）
   - 聊天交互：
     - 用户输入问题并发送
     - 用户消息即时显示在对话中
     - AI 回复显示为聊天消息气泡
     - 加载态：等待 AI 时显示“思考中…”
     - 错误态：网络/接口失败时给出友好提示（不暴露敏感信息）
   - 对话规则：
     - AI 主持人只输出：**是 / 否 / 无关**
     - 不解释、不剧透（不得输出与“汤底文本”相关的具体内容）
   - 操作按钮：
     - “查看汤底” -> 跳转汤底页面
     - “结束游戏” -> 返回大厅并重置本局状态

3. **汤底页面（Result）**
   - 展示故事标题
   - 展示完整 **汤底（bottom）**（揭晓真相的仪式感排版与动效）
   - 可选展示：玩家对话历史（帮助复盘推理）
   - “再来一局” -> 返回大厅并允许选择新故事

### 4.2 后续可做（非 MVP）
- 用户登录与个人游戏记录保存
- 排行榜 / 成就系统（例如：最短局数、最优提问次数）
- 自定义海龟汤：允许上传或编辑汤面/汤底并自动生成故事
- 多人对战或观战模式

## 5. 业务规则与约束
- **汤底安全边界**：
  - `bottom` 只由后端/AI 侧持有，前端只接收并展示“汤面”与“最终结果页面的汤底”（以结果页方式受控输出）
  - 防止在前端代码或网络请求中泄露汤底全文
- **AI 输出合规**：
  - 最终只允许三种文本之一：`是` / `否` / `无关`
  - 若模型输出不合规：后端进行校验并触发兜底策略（例如二次约束重试或降级输出）
- **可用性**：
  - AI 接口超时或失败时，前端给出清晰的错误提示，并允许用户重试

## 6. 界面设计要求（游戏官网风：深蓝 + 青蓝霓虹，科幻悬疑）
- 整体风格：神秘悬疑、深蓝色调背景，青蓝/靛蓝霓虹作为强调色
- 组件质感：玻璃拟态（半透明 + 边框 + backdrop blur），阴影克制但有层次
- 视觉重点：
  - 主标题微发光
  - 故事卡片 hover 有轻微上浮与霓虹边框增强
  - 聊天气泡区分用户/AI：颜色与对齐规则清晰
  - 揭晓页强调排版与淡入/轻微动效
- 响应式：
  - 移动端可正常显示标题、规则与聊天输入
  - 输入框与按钮不遮挡、可点击、可滚动

## 7. 技术栈与实现建议
- 前端：
  - React + TypeScript + Vite
  - Tailwind CSS（用于风格与布局统一）
  - 路由：React Router（后续实现 `Home / Game / Result`）
- 后端（建议必做，保护 Key 与实现输出校验）：
  - Node.js + Express
  - 提供 `POST /api/chat`：封装 AI 调用并校验输出
  - 使用环境变量管理 API Key，不在前端暴露
- AI：
  - DeepSeek / 其他大模型 API（本项目可选 DeepSeek）

## 8. 接口与数据约定（概念级）
- 故事数据模型（概念）：
  - `Story { id, title, difficulty, surface, bottom }`
  - MVP 展示 `surface`；揭晓时展示 `bottom`
- AI 对话接口（概念）：
  - `POST /api/chat`
  - 入参：`storyId`、`question`（可选：对话 history 摘要或数组）
  - 出参：`answer`（只允许是/否/无关）

## 9. 验收清单（Day 11 前尽量做到）
- 能进入大厅并看到故事卡片
- 能进入游戏页看到汤面 + 对话输入框
- 能发送问题并得到 AI 回复（且回复严格为“是/否/无关”）
- 能点击“查看汤底”并展示正确汤底（不泄露汤底到非揭晓页）
- “再来一局/结束游戏”能正常返回并重置状态
- 移动端布局正常、按钮可用
